## 二、回测策略

### 2.1 回测参数

| 参数 | 设置 |
|------|------|
| **选股比例** | 前5%（约25只） |
| **持仓时间** | 最少5天 |
| **止损机制** | -8%强制止损 |
| **交易成本** | 0.13%单次（滑点0.1% + 手续费0.03%） |
| **回测区间** | 2025-01-02 ~ 2025-08-29 |

### 2.2 交易逻辑

```
T日：选股（预测得分前5%）
T+1日：开盘买入
T+6日：开盘卖出（持有5天）
或：触发-8%止损时提前卖出
```

### 2.3 数据穿越检查

✅ **无数据穿越**
- 训练集：2020-01-02 ~ 2024-08-30
- 回测集：2025-01-02 ~ 2025-08-29
- 间隔：4个月（无重叠）

## 六、问题与改进

### 6.1 当前问题

| 优先级 | 问题 | 说明 |
|--------|------|------|
| 🔴 高 | 风险过高 | 最大回撤-25% vs 指数-10% |
| 🔴 高 | 缺少大盘过滤 | 未在熊市减仓 |
| 🟡 中 | 胜率偏低 | 49.4%接近随机 |
| 🟡 中 | 止损触发多 | 285笔(7.3%)触发-8%止损 |
| 🟢 低 | 持仓时间 | 平均4.9天，未达5天目标 |
| 🟢 低 | 股票名称 | 400只显示"股票xxx" |

### 6.2 改进建议

1. **添加大盘过滤**（MA20/MA60趋势判断）
2. **优化止损逻辑**（移动止损：最高价回撤8%）
3. **调整选股阈值**（从5%调到3%或7%）
4. **严格持仓时间**（确保满5天再卖出）

## 九、结论

**V2模型特点**：
- ✅ 排除了数据穿越特征，更稳健
- ✅ 考虑了交易成本，收益更真实
- ✅ 有止损保护，控制极端风险
- ❌ 仅勉强跑赢大盘（+0.98%）
- ❌ 风险远高于指数（回撤-25% vs -10%）

**适用场景**：
- 适合作为**基础策略**，需进一步优化
- 不适合**单独使用**，风险收益比不佳
- 建议**结合大盘过滤**和**移动止损**改进
