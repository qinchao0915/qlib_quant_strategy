## 二、V4核心改进

### 2.1 Alpha标签（超额收益预测）

**计算公式**:
```
Alpha = 股票未来5日收益 - CSI500指数未来5日收益
```

**实现代码**:
```python
# 计算指数收益
index_return = index_close.shift(-5) / index_close - 1

# 计算股票收益
stock_return = stock_close.shift(-5) / stock_close - 1

# Alpha标签
alpha_label = stock_return - index_return
```

**统计结果**:
- 均值：0.1300%
- 标准差：5.8279%
- 说明：Alpha分布较分散，预测难度大

### 2.2 市值中性化

**方法**：截面线性回归去除市值影响
```python
# 回归模型
feature = β * log(市值) + α + 残差

# 使用残差作为中性化特征
feature_neutral = residual
```

**中性化特征**（12个）:
- 动量特征：return_5d, return_10d, return_20d, return_60d
- 波动率：volatility_5d, volatility_10d, volatility_20d
- 技术指标：rsi_6, rsi_12, rsi_24, macd, macd_signal

**处理结果**:
- 训练集：13,310个特征值中性化
- 验证集：2,618个特征值中性化

### 2.3 LambdaRank排序学习

**原理**：直接优化排序质量（NDCG），而非数值准确性

**配置**:
```python
LGBMRanker(
    objective='lambdarank',
    metric='ndcg',
    ndcg_at=[5, 10, 20],
    learning_rate=0.02,
    n_estimators=500,
    label_gain=[0, 1, 3, 7, 15]  # 5个等级的增益
)
```

**标签转换**：
- 连续Alpha → 5个等级（0-4）
- 使用分位数分桶：qcut(5)

**训练结果**:
- 早停轮数：24轮
- 最佳NDCG@5：0.4194
- 最佳NDCG@10：0.4007

**可能原因**:

## 六、关键经验教训

### 6.1 Alpha预测的难度
- 预测相对收益比绝对收益更难
- 需要更复杂的特征和模型
- 可能需要更多数据

### 6.2 市值中性化的权衡
- 去除了市值因子暴露
- 但也可能去除了有效信息
- 需要平衡中性化和预测能力

### 6.3 排序学习的特点
- 优化NDCG ≠ 优化IC
- 更适合选股场景
- 需要配合TopN%评估

### 6.4 模型评估的多维度
- 不能只看IC
- 需要结合Rank IC、Top5%、NDCG
- 最终要看回测效果
